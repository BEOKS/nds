# Claude Opus 4.5 vs GPT-5.2 xhigh 모델 비교

## 배경
- 회사에서 Claude Opus 4.5 모델 지원
- 기존에는 Codex로 GPT-5.2 xhigh 모델 사용

## 모델별 특성 비교

| 항목 | Claude Opus 4.5 | GPT-5.2 xhigh |
|------|-----------------|---------------|
| 속도 | 몇 배 빠름 | 느림 |
| 응답 스타일 | 장황함 | - |
| 환각 현상 | 발생 (존재하지 않는 테이블 참조 등) | 거의 없음 |
| 복잡한 레거시 코드 | 환각 수정 공수 발생 | 신뢰성 높음 |

## 환각 현상 사례
- 존재하지 않는 테이블을 참조하여 주석 작성
- 가비아 레거시처럼 도메인, 비즈니스 로직, 코드가 복잡한 프로젝트에서 특히 문제

## 권장 사용 전략

### GPT-5.2 xhigh 사용 시점
- 개발 계획서 작성
- 코드 검수/리뷰
- 복잡한 도메인 분석

### Claude Opus 4.5 사용 시점
- 실제 코드 작성
- 빠른 반복 작업
- 단순 구현 작업
- 신규 프로젝트나 PoC 등 기존 제약조건에 얽매이지 않는 작업 (GPT보다 적합)

## 결론
- 둘 중 하나만 선택한다면 GPT-5.2 (신뢰성)
- 하이브리드 접근: 계획/검수는 GPT, 코드 작성은 Claude
- Claude 사용 시 컨텍스트 사전 로딩 및 즉시 검증 필요
