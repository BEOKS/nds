# 병렬 에이전트 워크플로우 관리

## 배경

Claude Code, Codex 같은 에이전트를 한 번에 하나만 쓰는 게 아니라 여러 프로젝트에서 여러 주제로 병렬 실행하는 경우가 많다. 변경사항 확인이 빈번해지면서 효율적인 워크플로우 관리가 필요해졌다.

## 현재 최선의 방법: tmux + helix

### tmux 활용

하나의 터미널을 필요에 따라 여러 패널로 분할해서 작업하는 것이 가장 효과적이다.

- 여러 에이전트 세션을 동시에 모니터링 가능
- 한눈에 전체 진행 상황 파악
- 패널 간 빠른 전환

### helix 에디터 활용

- 공식 사이트: https://helix-editor.com/
- 코드 변경사항을 터미널에서 바로 확인 가능
- LSP 내장으로 함수/변수 참조 기능 제공
- IDE로 작업 디렉토리 이동하는 번거로움 해소

## IDE(IntelliJ, VSCode) vs 터미널 기반 도구

### IDE가 여전히 필요한 경우

- Bean 등록, 시스템 프로퍼티 검색 (프레임워크 통합 기능)
- 브레이크포인트 기반 디버깅

### 터미널 기반으로 대체 가능한 경우

- 파일 변경사항 확인
- 코드 검색/탐색
- 함수/변수 참조 추적

## 디버깅 대안

1. **log.debug 활용**: 간단한 디버깅은 로그로 충분
2. **스캐폴딩 테스트 코드**: Codex로 빠르게 테스트 코드 작성해서 검증

## 결론

- **모니터링**: tmux 패널 분할
- **코드 확인**: helix (LSP 내장)
- **심화 작업**: IDE 병행 (디버깅, 프레임워크 통합 기능)

병렬 에이전트 실행 시 컨텍스트 스위칭 비용을 줄이는 것이 핵심이다.
