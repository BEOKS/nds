# Board Resolver: 보안포털 디버깅 워크플로우 자동화

**태그**: `AI`, `자동화`, `디버깅`, `워크플로우`, `Claude Code`

## 핵심 요약

> 수동으로 6단계를 거쳐야 했던 디버깅 과정이 `/board-resolver` 한 줄로 자동화되었다.

## 기존 디버깅 워크플로우 (수동)

보안포털에서 이슈 해결 시 거쳐야 했던 과정:

```
1. Mattermost 보드 읽기 → 이슈 내용 파악
2. 코드베이스 검색 → 관련 코드 찾기
3. 정책 확인 → 기존 정책 위배 여부 검사
4. 기획 확인 → 정책 추가/변경 필요 시 문의
5. 코드 수정 → 해결방안 구현
6. 테스트 작성 → 일일이 테스트 코드 작성
```

### AI 도입 초기 (Codex, Claude Code)

| 단계 | AI 활용 | 수동 작업 |
|------|---------|-----------|
| 보드 읽기 | ❌ | 직접 읽고 정리 |
| 코드 검색 | ❌ | 직접 검색 |
| 정책 확인 | ❌ | 직접 확인 |
| 기획 확인 | ❌ | 직접 문의 |
| 코드 수정 | ✅ | AI 도움 |
| 테스트 작성 | ✅ | AI 도움 |

**문제점**: 코드 작성만 도움받고, 나머지 5단계는 여전히 수동

## 해결책: board-resolver 스킬

### 자동화된 워크플로우

```
┌─────────────────────────────────────────────────────────────┐
│  /board-resolver <mattermost-url>                           │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Phase 1: 이슈 정보 자동 수집                                │
│  - Mattermost API로 보드/카드 정보 추출                      │
│  - 제목, 설명, 속성값, 첨부 콘텐츠 자동 파싱                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Phase 2: 코드베이스 자동 분석                               │
│  - 이슈 키워드 추출 (에러 메시지, 기능명)                    │
│  - Grep/Glob으로 관련 코드 자동 검색                         │
│  - 의존성 및 호출 관계 분석                                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Phase 3: 정보 충분성 검증 (반복 루프)                       │
│  - 정책 위배 여부 자동 검사                                  │
│  - 부족한 정보 → 사용자에게 질문 생성                        │
│  - 기획 확인 필요 시 → 정리된 질문 제시                      │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Phase 4-6: 해결방안 도출 → 구현 → 테스트                   │
│  - 점수화된 해결방안 제시                                    │
│  - 선택된 방안 자동 구현                                     │
│  - 테스트 코드 자동 작성 및 실행                             │
└─────────────────────────────────────────────────────────────┘
```

### 자동화 전후 비교

| 단계 | Before | After |
|------|--------|-------|
| 보드 읽기 | 수동 (5-10분) | 자동 (10초) |
| 코드 검색 | 수동 (10-30분) | 자동 (30초) |
| 정책 확인 | 수동 (5-15분) | 자동 검사 |
| 기획 확인 | 수동 문의 | 자동 질문 생성 |
| 코드 수정 | AI 도움 | AI 자동 구현 |
| 테스트 작성 | AI 도움 | AI 자동 작성 |

**결과**: 1-2시간 → 10-20분 (체감 시간 기준)

## 핵심 설계 원칙

### 1. 정보 수집 자동화

```python
# mattermost_cli.py board-card --card-url <URL>
# → 카드 제목, 설명, 속성, 첨부파일 자동 추출
```

### 2. 정책 검증 루프

기존 정책을 위배하는지 **자동으로 검사**하고, 추가 검토가 필요한 경우 **정리된 질문**을 생성:

```
## 추가 정보 요청

해결방안을 정확히 도출하기 위해 다음 정보가 필요합니다:

### 필수 정보
1. [질문] - 필요 이유: [이유]

### 선택 정보
2. [질문] - 필요 이유: [이유]
```

### 3. 점수 기반 의사결정

| 항목 | 가중치 | 역할 |
|------|--------|------|
| 원인 적합도 | 40% | 분석된 원인과 직접 연관 |
| 구현 복잡도 | 25% | 수정 범위 최소화 |
| 부작용 위험 | 20% | 다른 기능 영향도 |
| 테스트 용이성 | 15% | 검증 가능성 |

## 교훈

### AI 도구의 진짜 가치

- **코드 작성**만 자동화하면 효과 제한적
- **전체 워크플로우**를 자동화해야 실질적 생산성 향상
- 인간은 **의사결정**에만 집중

### 스킬 설계 원칙

1. **엔드-투-엔드**: 시작부터 끝까지 자동화
2. **반복 루프**: 정보 부족 시 자동으로 질문 생성
3. **점수화**: 객관적 기준으로 해결방안 순위화
4. **인간 개입 최소화**: 선택만 하면 나머지는 자동

## 결론

- AI 코딩 도구는 **코드 작성 보조**에서 **워크플로우 자동화**로 진화
- 반복적인 수동 작업을 스킬로 캡슐화하면 생산성 급상승
- 인간의 역할: 판단과 선택, AI의 역할: 실행과 검증
